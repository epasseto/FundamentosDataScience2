\hypertarget{foi-recebido}{%
\paragraph{Foi recebido:}\label{foi-recebido}}

\begin{itemize}
\item
  Um arquivo em formato \textbf{.csv} contendo dados de Twitagem do
  grupo, tendo já sido feito trabalho de elaboração sobre ele
\item
  nomes de cães foram capturados
\item
  campos de uma maneira geral já apresentavam alguma formatação
  demonstrando os objetivos a serem cumpridos
\item
  Um caminho a um conjunto de fotos a serem baixadas
\item
  neste foi feito um trabalho de \textbf{Machine Learning} a fim de
  interpretar, em cada foto, qual o cão que a aprecia
\item
  acompanhou um dataset onde é possível extrair o caminho para cada foto
\item
  A indicação do endereço do grupo no \textbf{Twitter}
\end{itemize}

\hypertarget{problemas-identificados}{%
\paragraph{Problemas identificados:}\label{problemas-identificados}}

O arquivo principal, o \textbf{.csv} aparentava melhor
\textbf{qualidade} do que o que posteriormente se percebeu:

\begin{itemize}
\item
  Problemas de qualidade identificados
\item
  o campo \textbf{tweet\_id} encontrava-se formatado como Inteiro Longo
  e não como String de Inteiros
\item
  o campo \textbf{timestamp} não seguia o formato Date
\item
  o campo \textbf{name} continha muitos registros de nomes inválidos
\item
  os campos \textbf{puppo}, \textbf{pupper}, \textbf{doggo} e
  \textbf{floofer} encontravam-se incompletos
\item
  os campos \textbf{rating\_numerator} e \textbf{rating\_denominator}
  continham entradas inválidas
\item
  linhas de registro de \textbf{retwitagens}, inúteis para o projeto
\item
  linhas de registo de \textbf{retwiter}, inúteis ao projeto
\item
  linhas de registro sem fotografias, inúteis ao projeto
\end{itemize}

O arquivo baixado do Twitter, também apresentava alguns problemas de
\textbf{qualidade}, a saber:

\begin{itemize}
\item
  Problemas de qualidade identificados
\item
  o campo \textbf{id} possuía nome diferente de \textbf{tweet\_id}
\item
  o campo \textbf{id} apresentava-se formatado como Inteiro Longo, como
  o anterior
\item
  o campo \textbf{retweet} estava formatado como Float, sendo Inteiro
\item
  o campo \textbf{favorites} estava formatado como Float, sendo Inteiro
\end{itemize}

Tendo como referência o arquivo \textbf{.csv}, sendo ele o objeto deste
trabalho, alguns problemas de \textbf{tidyness} foram identificados, a
saber:

\begin{itemize}
\item
  o campo \textbf{retweet} se encontrava no arquivo vindo do servidor
  Twitter
\item
  o campo \textbf{favorites} se encontrava no mesmo arquivo descrito
  acima
\item
  o campo \textbf{link} com a foto dos cães, se encontrava no arquivo
  vindo do trabalho de \textbf{Machine Learning}
\end{itemize}

\hypertarget{metodologia-empregada}{%
\subsubsection{Metodologia Empregada}\label{metodologia-empregada}}

Foi usada a técnica baseada em \textbf{Iterações} de ciclos de
aquisição, verificação e correção, até em tese se obter dados perfeitos,
ou aperfeiçoados, como no caso. Ela é uma metologia baseada em três
etapas:

\begin{itemize}
\item
  \textbf{Gather}, na qual se \textbf{enlaça} ou se captura, a maior
  quantidade de dados novos potencialmente úteis ao projeto
\item
  \textbf{Assess}, na qual se \textbf{esmiúça} os dados capturados, na
  tentativa de encontrar possíveis erros ou inconsistências, propondo
  soluções
\item

\end{itemize}
